Gitのコマンド集

gitの作成: git init

ファイルをステージに追加: git add (ファイル名) or git add .

コミット: git commit "コミット名"

ステージとリポジトリの差分を表示してコミット: git commit -v

現在の状態を確認: git status

ワークツリーとステージとの差分を表示: git diff

ステージとリポジトリとの差分を表示: git diff --staged

コミット履歴の追加: git log

リポジトリ、ディレクトリからファイルを削除: git rm（ファイル名）

リポジトリからは削除、ディレクトリには残す: git rm --cashed（ファイル名）

ファイルの名前を変更: git mv (元のファイル名) (変えたいファイル名)

リモートリポジトリを新規追加する: git remote add origin

リモートリポジトリへ送信する: git push (リモート名)（ブランチ名）

マスターへ送信:git push origin master

ファイルの変更を戻す: git checkout --(ファイル名　or ディレクトリ名))

ステージした変更を取り消す: git reset HEAD または　git reset HEAD　(ファイル名)

直前のコミットをやり直す（プッシュしたファイルは適応外でローカルのみで使う）: git commit --amend

リモートを表示する: git remote -v

リモートリポジトリを新規追加する: git remote add （リモート名）（リモートURL）

リモートから情報を取得する: git fetch (リモート名)

リモートから情報取得してマージする: git pull (リモート名)（ブランチ名）

リモートの詳細情報を表示する: git remote show (リモート名)

リモートを変更: git remote rename (旧リモート名)(新リモート名)

リモートを削除する: git remote rm リモート名

変更履歴をマージする: git marge (ブランチ名)

変更履歴をマージする: git marge (リモート名/ブランチ名)

コンフリクト事故の防止

①複数人で同じファイルを変更しない

②pullやmergeする前に変更中の状態を無くしておく(commitやstashをしておく)

③pullする時は、pullするブランチに移動してからpullする

ブランチを変更する: git branch -m (ブランチ名)

ブランチを削除する: git branch -d (ブランチ名)

プルリクエストの手順 ①git pull origin master : masterブランチを最新に更新 ②git checkout -b ブランチ名 : ブランチを作成する ③ファイルを変更する ④　git commit -v : 変更をコミット ⑤git push リモート名　ブランチ名 : Githubへプッシュ ⑥github上での操作 : githubプルリクエストを送る ⑦コードレビュー ⑧プルリクエストをマージ⑨ブランチを削除

リベースで履歴を整えた形で変更を統合する: git rebase (ブランチ名)

プルのリベース型: git pull --rebase (リモート名) (ブランチ名)

複数のコミットをやり直す: git rebase -i (コミットID） git rebase -i HEAD~3

タグの一覧を表示する: git tag

作業を一時的に避難させていく: git stash

stashの履歴を見る: git stash list

最新の作業を復元する: git stash apply

ステージの状況も復元する: git stash apply --index

特定の作業を復元する: git stash apply (スタッシュ名)

最新の作業を削除する: git stash drop

特定の作業を削除する: git stash drop (スタッシュ名)

全作業を削除する: git stash clear
